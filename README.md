# 🍕 Pizza Sales End-to-End Analysis

## 🎯 Executive Summary
This project analyzes a year's worth of sales data from a busy pizza restaurant to uncover peak operational hours, customer purchasing habits, and revenue drivers. The core objective was to move from raw, unorganized transaction data into actionable business intelligence that can help a restaurant manager optimize staffing levels and inventory control.

## 🛠️ Tools Used
- **SQL Server:** For data exploration, cleaning, aggregation, and extracting core business KPIs.
- **Power BI:** For building an interactive, dark-themed operational dashboard.

## 📊 Key Insights & Business Recommendations
- **Staffing Optimization:** Sales volume peaks significantly on Friday and Saturday evenings, specifically between 5:00 PM and 8:00 PM. *Recommendation: Increase kitchen and delivery staffing by 20% during these weekend windows to improve order fulfillment speed.*
- **Inventory Control:** The "Classic" pizza category contributes the highest overall volume to sales, while specific large pizzas drive the highest revenue. *Recommendation: Ensure ingredient stock levels for classic pizzas are filled to capacity ahead of the Thursday-Friday rush to avoid stockouts.*

## 📂 Project Structure
- `pizza project (sql ).sql`: Contains all the data extraction, analysis, and KPI queries.
